# SOSterm

Terminal emulator built from scratch by SOS Tech Services.

**Production version is C++/Qt5 (v2.x)** using QTermWidget for terminal emulation. This is what gets built, released, and deployed to clients via the update server.

The Python/PyQt5 code (v1.x) is the original prototype kept for reference/development only — it is NOT shipped or deployed.

Note: The project directory is still named `fterm` (the pre-rebrand name). All code, branding, and references now use **SOSterm**.

## Build & Run Commands

### C++ version (production — v2.x)

```bash
# Prerequisites: Qt5 (Widgets, Network, Svg), qtermwidget5, CMake >= 3.16
cd cbuild
cmake ..
make -j$(nproc)
# Binary: cbuild/SOSterm
```

## Deployment

### Install (requires root)

```bash
sudo ./install.sh
```

Installs to `/opt/SOSterm`, creates symlink at `/usr/local/bin/SOSterm`, installs desktop file and icons. The install script auto-detects whether to use the C++ build (`cbuild/SOSterm`) or a packaged binary.

### Release tarball

```bash
./release.sh
```

Builds a portable, self-contained release from the C++ binary. Bundles all shared libraries, Qt platform plugins, and QTermWidget data. Outputs `dist/SOSterm-<version>.tar.gz` and `dist/latest.json` for the update server.

Version is extracted from `cbuild/version.h` (generated by CMake from `src/version.h.in`).

Deploy artifacts to update server:
```bash
cp dist/SOSterm-<version>.tar.gz /mnt/sos-updates/fterm/
cp dist/latest.json /mnt/sos-updates/fterm/
```

### Uninstall

```bash
sudo /opt/SOSterm/uninstall.sh
```

## Key Subsystems

### Terminal rendering (Python)

`terminal_widget.py` is the largest and most complex module. It is a custom QWidget (not using QTermWidget) that:

- Uses **pyte** `Screen` + `ByteStream` for VT100/xterm escape sequence parsing
- Renders the terminal grid character-by-character via QPainter
- Handles ANSI colors 0-15, xterm-256, and 24-bit truecolor (`\e[38;2;r;g;b`)
- Supports block/underline/bar cursor styles with blinking
- Implements mouse-based text selection with word and line select modes
- Provides scrollback buffer navigation (mouse wheel, Shift+PgUp/PgDn)
- Detects and renders clickable URLs
- Sanitizes escape sequences that pyte cannot handle (DCS, APC, PM, CSI sub-parameters)

### PTY I/O (`terminal_process.py`)

Uses a QThread (`PtyReaderThread`) with a drain loop pattern: after `select()` indicates data is ready, it reads in a tight loop with `select(..., timeout=0)` to coalesce many small kernel PTY reads into a single large emission. This reduces cross-thread signal overhead by 100-200x during rapid output.

### SSH management

Sessions are stored as JSON in `~/.config/SOSterm/ssh_sessions.json`. SSH connections open a new tab running the `ssh` command (built by `SSHSession.build_command()`). The sidebar supports groups, quick-connect, and import from `~/.ssh/config` and Remmina `.remmina` files.

### Theme system

Themes define terminal colors (bg, fg, cursor, selection, 16 ANSI colors) plus UI chrome colors (tab bar, status bar, menus, borders). `get_app_stylesheet()` generates a complete Qt stylesheet from a theme. The `Settings.settings_changed` signal triggers live theme switching.

### Update checker

Checks `https://sos-tech.ca/updates/fterm/latest.json` in a background QThread. Auto-checks on startup with a 24-hour cooldown; manual checks always run. The `latest.json` contains version, download URL, changelog, SHA256, and size.

## Critical Patterns

- **Signal-driven architecture**: Settings changes propagate via `Settings.settings_changed(key, value)`. MainWindow listens and dispatches to theme/font/padding update methods. The signal carries both the key name and value.
- **Fusion style forced**: `app.setStyle(QStyleFactory.create("Fusion"))` in `main.py` prevents system dark themes from bleeding through.
- **Private member convention**: All internal attributes use underscore prefix (`self._settings`, `self._tab_manager`). Public API uses non-prefixed methods.
- **Session restore**: On close, `SessionManager` saves open tabs (CWD, SSH session ID, active index) to `session.json`. On startup, tabs are recreated. The `-n` flag skips restore.
- **High-DPI**: Enabled via `Qt.AA_EnableHighDpiScaling` and `Qt.AA_UseHighDpiPixmaps`.
- **CLAUDECODE env var**: `terminal_process.py` strips the `CLAUDECODE` environment variable so nested Claude Code sessions work.
- **Icon resolution**: Checks local `resources/SOSterm.svg` first, falls back to `/opt/SOSterm/resources/SOSterm.svg` for installed deployments.
- **Tab auto-close**: When a shell process exits, its tab is automatically closed. When the last tab closes, the window closes.

## Data Storage

| Path | Content |
|------|---------|
| `~/.config/SOSterm/settings.json` | User preferences (theme, font, shell, scrollback, cursor style, window geometry, etc.) |
| `~/.config/SOSterm/session.json` | Saved terminal session (open tabs, CWDs, active tab index) |
| `~/.config/SOSterm/ssh_sessions.json` | SSH session and group definitions |
| `/opt/SOSterm/` | Installed application (binary, resources, colorschemes, uninstall script) |
| `/usr/local/bin/SOSterm` | Symlink to `/opt/SOSterm/SOSterm` |
| `/usr/share/applications/SOSterm.desktop` | Desktop menu entry |
| `/usr/share/icons/hicolor/` | App icons (SVG scalable + generated PNGs at 16-256px) |

## Conventions

- **Python**: snake_case for functions/variables, PascalCase for classes. PyQt5 signals use camelCase (Qt convention). Type hints used sparingly (mainly in `ssh_session_store.py`).
- **C++**: PascalCase for classes, camelCase for methods/members. Qt naming conventions throughout. C++17 standard.
- **File naming**: Python uses snake_case filenames. C++ uses lowercase-no-separator filenames matching the class name.
- **Organization**: SOS Tech Services (`sos-tech.ca`). GitHub: `frankkahle/SOSterm`. Brand color: `#0047FF`.
- **Dependencies (Python)**: PyQt5 >= 5.15, pyte >= 0.8, ptyprocess >= 0.7, wcwidth >= 0.2.
- **Dependencies (C++)**: Qt5 (Widgets, Network, Svg), qtermwidget5, CMake >= 3.16, C++17.
- **Build outputs**: PyInstaller builds go to `dist/`. C++ builds go to `cbuild/`. Release tarballs go to `dist/SOSterm-<version>.tar.gz`.
